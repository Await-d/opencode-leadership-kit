# .agentdocs/workflow/260317-提示词格式分析.md

## 任务概述
分析现有 OpenCode + oh-my-opencode 提示词/命令格式，并判断用户提供的 ultrawork 模板是否可直接采用或需分层适配。

## 当前分析
- 官方格式来源：OpenCode Agents/Commands 文档。
- 本地格式来源：opencode.json、.opencode/agents、.opencode/commands、oh-my-opencode.json。
- 目标是明确“哪些层可以承载超长规则、哪些层必须保持简洁”。

## 方案设计
以“层级职责”拆分超长模板：
1) Leader：全局编排/委派/验证理念
2) 子代理：角色职责与输出契约
3) Command：流程入口与强制步骤
4) Plugin prompt_append：轻量路由与标签

## 实施计划
### Phase 1: 约束与现状收集
- [x] T-01：收集官方与本地的 agent/command 格式约束
- [x] T-02：检查 leadership-kit 现有 prompts，并对照 ultrawork 模板关键段落

### Phase 2: 兼容性结论
- [x] T-03：给出“可用/需拆分/不应复制”的最终判断与分层建议

## 备注
- 仅做分析与结论，不修改任何提示词或配置文件。
